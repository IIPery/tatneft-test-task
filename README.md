# Metrics API
* Хусаинов Искандер. Тестовое задание

## Описание
#### Цель: 
* Простой REST API для управления метриками с минимальным набором функций и настроенной админ панелью.
#### Требования:
* Приложение должно запускаться в docker-compose
* Приложение должно быть реализовано на DRF
* База данных PostgreSQL
* Redis (кэш, брокер Celery)
* Простая JWT авторизация
* Настроено создание суперпользователя, а затем добавление пользователей через админ панель
* Настроить кэширование списка записей метрик в метрике, при добавлении записи инвалидировать кэш
* Настроить Celery Beat задачу, которая создает «фейк отчет». В отдельной директории в корне проекта создает, а затем обновляет текстовый файл раз в 2 минуты с содержанием общего количества метрик и записей в базе данных.
* Настроить админ панель для управления Metric, MetricRecord, Tag
* Эндпоинт на создание записи метрики покрыт тестами
#### Дополнительно
* Добавлена команда для создания суперюзера, обычного юзера и тестовых объектов при запуске проекта
* Добавлен swagger http://localhost:8000/api/swagger/
* Добавлен github CI


## Установка проекта

1) Склонировать проект:
    ```shell
    mkdir khusainov_test_task
    cd ./khusainov_test_task
    git clone https://github.com/IIPery/tatneft-test-task.git
    cd tatneft-test-task     
    ```

2) Копирование переменных окружения:
    ```shell
    cp .env.example .env && sed -i "s#SECRET_KEY=.*#SECRET_KEY=$(openssl rand -hex 32)#" .env
    ```

3) Запуск проекта:
    ```shell
   docker compose up --build
    ```

4) Админка/Swagger:
* http://localhost:8000/admin/
* http://localhost:8000/api/swagger/
